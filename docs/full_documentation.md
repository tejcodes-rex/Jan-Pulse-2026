# Problem Statement: The "Silent" Exclusion
    
While Aadhaar coverage is near-universal/saturation, "Dormancy" is the new exclusion. millions of residents hold valid IDs but represent "Dead Signals" in the databaseâ€”no updates, no authentications, no linkage to services.

# The Jan-Pulse Approach
We shifted from "Stock Analysis" (Who has Aadhaar?) to "Flow Analysis" (Who is using Aadhaar?). By analyzing update velocity (Demographic/Biometric), we triangulated three critical signals:
1. Ghost Children (School Dropouts)
2. Workforce Velocity (Migration Corridors)
3. Digital Dark Zones (Financial Exclusion)

# Technical Approach

# Technical Methodology: Map-Reduce & Triangulation

## 1. Map-Reduce Architecture
To process high-volume distributed CSVs without memory overflow:
- **Map**: Processed each file individually, standardizing dates (YYYY-MM) and scrubbing filename-based state labels.
- **Reduce**: Aggregated data chunks into condensed "Master" datasets using `pandas.melt` for schema normalization.

## 2. Triangulation Logic
We derived "Indices" rather than raw counts:
- **Dropout Risk**: Compared `Enrolment Cohort (Age 0-5)` vs `Biometric Updates (Age 5-17)`. A lag suggests school exclusion.
- **Migration Velocity**: High volume of Adult Demographic updates proxies labor movement.
